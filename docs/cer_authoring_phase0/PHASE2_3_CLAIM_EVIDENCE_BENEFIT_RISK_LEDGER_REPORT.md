# Phase 2.3 Claim-Evidence-Benefit-Risk Ledger Hardening Report

Date: 2026-05-10

## Conclusion

`PHASE2_3_ACCEPTED_FOR_CCD_READ_ONLY_VALIDATION`

AGG-004 / G38 recurrence was addressed by adding a deterministic Claim-Evidence-Benefit-Risk writer spine before CER chapter generation. The patch does not change G38 pass/fail criteria, does not change 1+6 agent roles/prompts, and does not change other G0-G38 gate criteria.

## Root Cause Addressed

Before this patch, conclusion-strength control was mostly represented as a SOTA-level guard artifact. The writer could still produce sections 4.7 and 5 without consuming a mandatory claim-level chain:

`claim -> PICO -> endpoint -> SOTA benchmark -> evidence -> gap/PMCF -> risk controls -> benefit-risk judgment -> allowed wording`

That left G38 vulnerable to recurrent failures because the guard was not sufficiently tied to the actual writer-stage conclusion logic.

## Implemented Changes

### 1. Claim-Evidence Matrix

New artifact:

- `claim_evidence_matrix`
- export: `claim_evidence_matrix.xlsx`

The matrix maps each claim to:

- PICO
- endpoint IDs
- SOTA benchmark IDs
- evidence IDs
- evidence strength
- support status
- gap/PMCF IDs
- conclusion strength
- writer instruction
- QA flag

### 2. Benefit-Risk Ledger

New artifact:

- `benefit_risk_ledger`
- export: `benefit_risk_ledger.xlsx`

The ledger maps each benefit/claim to:

- supporting evidence
- SOTA benchmark
- key risk IDs
- IFU/RMF risk-control summary
- residual gaps
- PMCF references
- overall benefit-risk judgment
- allowed wording
- not-allowed wording
- CER section use

### 3. Writer Conclusion Strength Guard

New artifact:

- `writer_conclusion_strength_guard`
- export: `writer_conclusion_strength_guard.xlsx`

The writer guard is generated from the claim-evidence matrix and benefit-risk ledger. It is also merged into `sota_conclusion_strength_guard`, so existing G38 can validate the strengthened guard without changing G38 criteria.

The generated writer guard uses controlled wording and avoids raw overclaim terms in CER prose.

### 4. CER Section Trace Map

New artifact:

- `cer_section_trace_map`
- export: `cer_section_trace_map.xlsx`

The trace map links each writer paragraph target to:

- claim IDs
- endpoint IDs
- SOTA benchmark IDs
- evidence IDs
- gap IDs
- PMCF IDs
- risk IDs
- benefit-risk IDs
- allowed wording strength
- guard row ID

### 5. Writer-Stage Enforcement

`write_cer_chapters()` now builds the ledger before chapter generation. Chapters 4.7 and 5 now include and consume:

- Claim-Evidence Matrix
- Benefit-Risk Ledger
- Writer Conclusion Strength Guard

If chapters already exist, the function can still generate the missing ledger artifacts without rewriting the existing chapters.

### 6. Artifact/Workbook Contract

The following keys are now part of `SharedAuthoringState`, `authoring_workbook.json`, and exportable XLSX artifacts:

- `claim_evidence_matrix`
- `benefit_risk_ledger`
- `writer_conclusion_strength_guard`
- `cer_section_trace_map`

The Phase 0 artifact consumption contract was extended to show producer/consumer lineage for the new Phase 2.3 artifacts.
