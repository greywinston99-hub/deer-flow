## Human Reviewability Criteria

Regenerated pilot CERs must satisfy all seven rules. A professional reviewer must be able to read and assess clinical validity without first filtering out system artifacts.

**R1 Device Description Source-Grounded.** Every 2.1 field has a source anchor (document and page reference) and extraction confidence. Fields without IFU data show `IFU source does not contain this information`, not `Not extracted from IFU source text; refer to subject device IFU for details`.

**R2 Intended Purpose Coherent.** 2.2 text matches locked device identity. No cross-domain clinical terms from other device families. Cardiac stabilizer report does not mention ablation. Plasma electrode report does not mention ureteroscopy.

**R3 SOTA Domain Correct.** 3.x clinical background discusses the correct clinical field for this device. Cardiac stabilizer discusses CABG and off-pump surgery. Plasma electrode discusses arthroscopic joint surgery. No urology in cardiac reports. No cardiac ablation in orthopedic reports.

**R4 Evidence Applicability Explained.** Each claim's evidence basis is explicit — which evidence items support it, from which sources, with what strength. Not template boilerplate. Not "clinical data partially support" applied uniformly to all claims regardless of support level.

**R5 Conclusion Respects Claim Constraints.** Summary and Chapter 5 language is consistent with claim_support_matrix. INSUFFICIENT claims are described as insufficient. MODERATE claims as moderate. No claim is described as "supported" when the matrix says INSUFFICIENT.

**R6 No Template Shell or Workflow Explanation Leakage.** CER body does not contain workflow instructions like "should be evaluated", "refer to IFU for details", "confirm with manufacturer". These are internal process notes, not submission content. CER body states what IS, not what SHOULD BE DONE.

**R7 No Internal System Language.** Zero occurrences of: Claude, DeerFlow, MCP, not_allowed, ALLOWED_USE_BLOCKED, score: 100, benchmark decision: PASS, cer_authoring_v1, AP/human CER writing paradigm, Generated by cer_authoring, Knowledge-base guidance. These belong in audit artifacts only.
