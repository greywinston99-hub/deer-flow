# B-G42-03: Per-Type Sufficiency Thresholds

- **Type**: Deterministic
- **Step**: Evidence Sufficiency Gate (G42, Step 16A)
- **Batch**: P0
- **Agent**: authoring-evidence-agent

## Input
- `claim_ledger` with claim_type
- `CLAIM_TYPE_SUFFICIENCY` configuration
- `pre_g42_claim_evidence_candidate_matrix`

## Output
- Per-type sufficiency judgment
- Failure pattern with type-specific repair route

## Per-Type Thresholds
| claim_type | Threshold | Rule |
|-----------|----------|------|
| clinical_benefit | min_pivotal: 1, min_supportive: 1 | SOTA comparable studies × quality distribution |
| intended_purpose | min_pivotal: 1, min_supportive: 1 | Clinical evidence in target population |
| safety | min_independent_sources: 2 | Multi-source cross-validation |
| performance | min_pivotal: 1 | Direct performance data |
| IFU_warning | min_rmf_coverage: 0.8 | RMF coverage ≥ 80% + GSPR trace |
| warning_contraindication | min_rmf_coverage: 0.8 | RMF/GSPR coverage |
| technical | min_supportive: 1 | Bench test or IFU spec |

## Decision Logic
1. Read `claim_type` for each claim from `claim_ledger`
2. Look up sufficiency framework from `CLAIM_TYPE_SUFFICIENCY`
3. For IFU_warning/warning_contraindication: check RMF presence + GSPR coverage, NOT PubMed hits
4. For safety: require ≥ 2 independent evidence sources from different databases
5. For clinical_benefit: require SOTA comparable study with quality distribution
6. Thresholds configurable; engineer confirms threshold table

## Checks
- IFU_warning not marked INSUFFICIENT due to missing PubMed evidence
- Safety claims validated against multiple independent sources
- All thresholds documented and configurable
- Engineer-confirmed threshold table
